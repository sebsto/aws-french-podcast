title: "AXA : l'IA agentique au service du développement logiciel"
description: "Dans cet épisode enregistré au Summit AWS Paris, Tanguy Combe, Lead Engineering Product Manager chez AXA Digital Commercial Platform, partage le retour d'expérience de son équipe de 70 personnes sur l'utilisation des agents IA dans le cycle de développement logiciel. AXA DCP, spécialisée dans le calcul de risque pour l'assurance, a progressivement adopté ces outils : d'abord un LLM local avec LM Studio et le plugin Continue, puis CodeWhisperer, Amazon Q Developer, et aujourd'hui Kiro. Tanguy détaille comment les agents interviennent à chaque étape du SDLC : brainstorming, rédaction de user stories, prototypage rapide pour valider des hypothèses avec les utilisateurs, génération de code en TDD, property-based testing, jusqu'à la maintenance et le diagnostic d'incidents. Il insiste sur les garde-fous indispensables : code review systématique par des humains, review des prompts pour comprendre le cheminement de pensée des agents, interdiction du accept all, et obligation pour les développeurs juniors d'expliquer le code généré lors des pull request reviews. La discussion couvre aussi la gestion de la fenêtre de contexte avec les steering rules et powers de Kiro, la mise en place d'une power factory partagée entre tous les développeurs, l'orchestration de plusieurs agents spécialisés via Kiro CLI et des workflows SpecKit, le contrôle des coûts en tokens intégré au FinOps, et les métriques DORA pour mesurer l'efficacité. Tanguy partage sa vision du nouveau rôle du développeur : moins d'écriture de code, plus de supervision d'agents, de compréhension du métier et de contrôle qualité."
