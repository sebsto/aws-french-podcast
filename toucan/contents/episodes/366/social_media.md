🤖 L'IA agentique au service du développement logiciel chez AXA

Tanguy Combe, Lead Engineering Product Manager chez AXA Digital Commercial Platform, partage le retour d'expérience de son équipe de 70 personnes sur l'utilisation des agents IA dans tout le cycle de développement.

AXA DCP, spécialisée dans le calcul de risque pour l'assurance, est passée du code completion avec LM Studio et CodeWhisperer à l'orchestration de plusieurs agents spécialisés via Kiro et Kiro CLI.

Les agents interviennent à chaque étape du SDLC :
→ Brainstorming et rédaction de user stories
→ Prototypage rapide pour valider des hypothèses avec les utilisateurs
→ Génération de code en TDD et property-based testing
→ Diagnostic d'incidents et maintenance

Les garde-fous mis en place :
→ Code review systématique par des humains
→ Review des prompts pour comprendre le raisonnement des agents
→ Interdiction du "accept all"
→ Les juniors doivent expliquer le code généré en PR review

Tanguy partage aussi la mise en place d'une "power factory" partagée entre tous les développeurs, l'orchestration multi-agents via SpecKit, le suivi des coûts en tokens intégré au FinOps, et les métriques DORA pour mesurer l'efficacité.

Sa vision : le développeur n'écrit plus le code, il supervise les agents, comprend le métier et contrôle la qualité.
